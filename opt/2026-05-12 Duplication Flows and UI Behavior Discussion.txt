📒Meeting Outline

Duplication flows and UI behavior

• Duplicate with data (checkbox selected)
  Team aligned that tooltip(s) and field disabling apply only when the user selects “Duplicate with data.” In this case, certain fields will be disabled and helper texts/tooltips shown per Figma. The run details tab should be visible even while the model remains in Draft after duplication with data.
• Duplicate without data (checkbox not selected)
  No UI/behavior change needed; it should continue to work exactly as it does today. No tooltips or disabled fields for this path.
• Tooltips and conditional disabling
  Tooltips are displayed only in the “duplicate with data” scenario to explain disabled fields and constraints. “Enable everything” clarification was corrected to mean enable per current behavior for “without data,” and apply restrictions for “with data.”

Run Details tab logic and backend considerations

• When to show Run Details
  Agreement that Run Details should be shown for duplicates created “with data,” even if the status is Draft. Do not rely on a “success” status to render Run Details for this scenario.
• Flag vs. inferred state
  Discussion on whether a backend/UI flag is required to control Run Details visibility and other UI states. Ashish proposed a flag for quick UI determination (lighter initial API payload) rather than waiting for full Run Details data. Counterpoint suggested data presence (copied tabs/data) and existing RPT Reference ID could implicitly indicate “with data.” Conclusion: treat this as an implementation detail between UI and API; create a backend story only if needed during implementation.
• RPT Reference ID usage
  RPT Reference ID will exist for both duplication paths. It alone is insufficient to distinguish “with data” vs “without data”; additional signal (flag or data presence) may be used to control Run Details/tab rendering.

User stories, automation, and refinement

• Current user stories
  Two small UI stories will proceed now; remaining items will be refined once Pete is back. Team to vote/confirm user story readiness and impacts.
• Automation impact
  Existing automation needs enhancement to cover the “duplicate with data” scenario and related UI conditions. Team acknowledged potential impact and will update tests accordingly.
• Backend story (conditional)
  If a flag is needed to control Run Details/tab visibility or persist “copied with data” state, a backend story will be created. For now, proceed with UI story and verify during implementation.
