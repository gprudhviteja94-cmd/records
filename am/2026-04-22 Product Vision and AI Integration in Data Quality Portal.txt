📒Meeting Outline

Product vision and platform scope

• Intern introductions and current focus
  Charlie and Anush introduced as interns (Jan–Jul). Current streams: Portal/UI, self-serve UI, and back-end automation. Upcoming stream: AI integration led by Acharya and Antarsh.

• End-to-end evolution to a platform
  Transition from server-run scripts (Corner Store) → Lumi framework → comprehensive DQI portal with UI, automation, and review workflows to make a “proper platform.”

• High-level portal dashboard
  Mock UI shows summary KPIs: tables covered, number of checks, open alerts, pass rate, total rules, volume trends, use-case filters, and a summarized open-alerts panel. Goal: enable teams to quickly extract metrics and status.

Alerts and rules management

• Alerts page parity with existing DQM
  Left-nav to Alerts with stats, filters (incl. date range), and alert list. Inline actions via three-dot menu to manage alerts (action panel not in mock; exists in current tool).

• Rules catalog and discovery
  Rules tab to browse existing checks by table or use case (e.g., transaction table). Supports search and visibility for team-specific check sets. Being built into DQI portal.

• Review and approval workflow
  New requests flow to Data Office review: approve/decline/modify. Upon approval, SFT pipeline feeds automation to generate SQL, create artifacts, and deploy after testing/validation.

New request intake and control-based guidance

• Structured request form
  Front-end captures table, column, check type (null/duplicate/threshold), and thresholds (e.g., 90–100). Submission triggers SFT to Lumi and back-end automation to generate/append SQL and push after validation.

• Historical performance vs. ML-suggested thresholds
  Control module shows:
  • Historical 6–12 month performance to validate proposed ranges.
  • ML-powered threshold suggestions (Quantro-related) tailored to historical behavior.
  Users can compare “given vs. suggested,” view insights (false-positive reduction, pass-rate lift, seasonality/variance), and select via “add to cart.”

• Real-time feasibility discussion
  Real-time computation is limited due to ACE–Lumi integration restrictions and library constraints. Options:
  • Batch or delayed report generation after executing BigQuery jobs.
  • Store requisite historical aggregates in ACE where possible.
  • Explore authenticated BigQuery access from ACE if policies permit; involves ASCORE review and leadership approvals.

AI-driven check recommendations

• LLM-assisted new checks generation
  Inputs: business context (docs, PPT, Excel), metadata, current checks, data profiling, vector embeddings. Prompting via ChatGPT chat UI (no API) to propose 3 nuanced checks with descriptions, expected pass/fail metrics, findings, and SQL implementations. Partners can select relevant checks to add to cart.

• Guardrails and cost considerations
  Avoid unnecessary queries on sensitive SD/card/member data; minimize repeated decryptions; add prompt- and agent-level validation before execution. Balance compute cost and necessity.

• AI access constraints and next steps
  Current allowance: ChatGPT chat and Custom GPTs for metadata/profiling context; no programmatic/API use. Need leadership alignment (discuss with Manju and Marcus) and table-owner approvals for what metadata may be shared even in internal tools. Execution remains in BigQuery; LLMs only produce query logic.

Anomaly detection and record-level insights

• Isolation Forest for outlier patterns
  Identify correlated variables aligned to business behavior; segment into clusters that reveal divergent trends (e.g., decreasing when normally increasing). Planned outputs:
  • Normal behavior graphs for last year.
  • 2–3 anomalous clusters with example customer trajectories.
  • Drill-down to record-level patterns to inform new checks.

• From insight to action
  If a cluster appears anomalous, users can add a corresponding check to cart, proceed to submission, and route into review/approval.

UI implementation feasibility and sharing

• Library and workspace constraints
  Q-Track space has restrictions on custom libraries; needs ASCORE review and leadership approval (Amex restrictions). Feasibility to be confirmed with Raj and platform owners.

• Mock-up access/sharing
  Replit sharing limited; files not consistently openable. Workaround: share annotated screenshots to guide self-serve/Q-Track UI improvements.
