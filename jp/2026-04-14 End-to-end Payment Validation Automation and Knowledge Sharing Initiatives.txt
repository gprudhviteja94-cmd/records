📒Meeting Outline

End-to-end Payment Validation Automation (SFTP → Workbasket → GSDB → Excel → S3 → Notification)

• Process kickoff and stakeholder communication
  The team will notify Ops, business stakeholders, and the migration team that the automated process has started. A standardized email, including the output Excel link, will be sent to all relevant groups.

• Multi-file ingestion from SFTP
  The system supports parallel processing of multiple files on the same day. It fetches all available files from SFTP Outbox (files then move to “send” folder), compiles the file list in a service bundle, downloads them, and parses required fields (account number, amount, payment date, check number).

• Dynamic workbasket creation and data loading
  For each input file, a dynamic (not static) workbasket is created. Parsed records are loaded into the workbasket after validating/connecting account numbers. The process then triggers a defined BPMN flow to handle all tasks.

• Per-task subprocess and data enrichment
  A subprocess executes for each task (e.g., 100 tasks → 100 subprocess calls). It retrieves card/account and payment details and calls GSDB: 
  • Read Payment API to get last 13 months of payment history per account.
  • Read Payment by Payment Identifier API to fetch detailed fields (check number, payment date, amount).

• Validation against source Excel
  The system matches transaction date, amount, and check number from the source Excel to GSDB data. Records are flagged Y if all fields match, N if mismatched.

• Result collation and output generation
  After all tasks complete, results for all accounts (e.g., ~110) are aggregated to create an Excel report. The Excel is uploaded to S3, and the shareable link is included in a notification email to business/client teams.

• Business follow-up on mismatches
  Business teams use the Excel to identify and investigate misplaced/missed payments per account and update accurate information as needed.

• Reusable technical patterns highlighted
  The use case demonstrates dynamic workbasket creation, SFTP file handling, BPMN-driven task processing, GSDB API integrations, Excel generation, S3 uploads, and standardized stakeholder notifications.
